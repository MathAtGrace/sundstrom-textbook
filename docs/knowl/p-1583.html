We have been implicitly using the fact that an integer cannot be both even and odd. There are several ways to understand this fact, but one way is through the Division Algorithm. When we classify an integer as even or odd, we are doing so on the basis of the remainder (according to the Division Algorithm) when the integer is “divided” by 2. If \(a \in \mathbb{Z}\text{,}\) then by the Division Algorithm there exist unique integers \(q\) and \(r\) such that

\begin{equation*} a = 2q + r\text{ and } 0 \leq r \lt 2\text{.} \end{equation*}

This means that the remainder, \(r\text{,}\) can only be zero or one (and not both). When \(r = 0\text{,}\) the integer is even, and when \(r = 1\text{,}\) the integer is odd.
